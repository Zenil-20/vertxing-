# Vertxing — Deployment (free, fast, Android PWA)

This deploys Vertxing so you can **install it on your Android phone** (as a PWA) and
make real calls — over HTTPS, on a 100%-free stack, with fast global media.

There is **no app store and no native build**. The app is a PWA: once it's online
over HTTPS, you "Install app" from Chrome and it behaves like a native app
(home-screen icon, full screen, background-call notifications).

---

## The stack (all free tiers)

| Piece | Service | Notes |
|---|---|---|
| Web (Next.js) | **Vercel** | Free, auto-HTTPS, global CDN |
| Media / SFU | **LiveKit Cloud** | ⭐ Fast global edge + TURN included — this is the "fast" part |
| Database | **Neon** (Postgres) | Free serverless Postgres |
| Redis | **Upstash** | Free serverless Redis |
| API (NestJS + sockets) | **Render** (Docker, free) | Supports WebSockets; **sleeps when idle** (see caveat) |

**Caveat — free API sleeps:** Render's free service spins down after ~15 min idle,
so the first call after a quiet period is slow and a background-push call can miss
(the server has to wake up). Two fixes:
- **Always-on & free:** run the API Docker image on an **Oracle Cloud "Always Free"
  ARM VM** (24/7, never sleeps). More setup — see the last section.
- **~$5/mo:** a small always-on instance (Render Starter / Fly.io / Railway).

Media quality is **free and fast regardless** thanks to LiveKit Cloud.

## 0. Push your code to GitHub

Vercel and Render deploy from a Git repo:

```bash
git init && git add . && git commit -m "Vertxing"
# create an empty GitHub repo, then:
git remote add origin https://github.com/<you>/vertxing.git
git push -u origin main
```

> `.gitignore` already excludes `node_modules`, `.next`, and `.env`. **Never commit
> real secrets** — they go in each host's dashboard.

## 1. LiveKit Cloud (media) — ~3 minutes

1. Sign up at **livekit.io** → create a project (pick a region near you).
2. Open **Settings → Keys** and copy:
   - **WebSocket URL** → `LIVEKIT_URL` (looks like `wss://your-project.livekit.cloud`)
   - **API Key** → `LIVEKIT_API_KEY`
   - **API Secret** → `LIVEKIT_API_SECRET`

That's it — TURN and global routing are handled for you.

## 2. Neon (Postgres)

1. Sign up at **neon.tech** → create a project/database named `vertxing`.
2. Copy the **pooled** connection string (ends in `-pooler...`, includes
   `?sslmode=require`) → `DATABASE_URL`.

## 3. Upstash (Redis)

1. Sign up at **upstash.com** → create a Redis database (region near your API).
2. Copy the **`rediss://`** (TLS) URL → `REDIS_URL`.

## 4. Generate the app secrets

```bash
# JWT + license secrets (run 3×, use a different value for each):
node -e "console.log(require('crypto').randomBytes(48).toString('base64url'))"

# VAPID keys for background calls (one pair):
node -e "console.log(require('web-push').generateVAPIDKeys())"
```

## 5. Deploy the API to Render

1. **render.com** → **New → Blueprint**, point it at your repo (it reads
   `render.yaml`), **or** New → **Web Service** → Docker, Dockerfile
   `apps/api/Dockerfile`, context `.`.
2. Set the environment variables (see `apps/api/.env.production.example`):
   `DATABASE_URL`, `REDIS_URL`, `LIVEKIT_URL`, `LIVEKIT_API_KEY`,
   `LIVEKIT_API_SECRET`, `VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT`,
   and (temporarily) `CORS_ORIGINS=https://localhost` — you'll fix it in step 7.
   - `PORT` is injected by Render — leave it unset.
   - `JWT_*`/`LICENSE_SECRET` can be auto-generated by the blueprint.
3. Deploy. On boot it runs `prisma migrate deploy` (creates all tables) and starts.
   Note the URL, e.g. `https://vertxing-api.onrender.com`. Check
   `https://vertxing-api.onrender.com/api/health` returns OK.

## 6. Deploy the Web to Vercel

1. **vercel.com** → **Add New → Project** → import your repo.
2. **Root Directory: `apps/web`** (important). Framework auto-detects Next.js;
   `apps/web/vercel.json` already sets the monorepo build (it builds the shared
   package first).
3. Add env var (Production): `NEXT_PUBLIC_API_URL = https://vertxing-api.onrender.com`
   (your Render URL). _This is baked at build time._
4. Deploy. Note the URL, e.g. `https://vertxing.vercel.app`.

## 7. Connect them (CORS)

1. Back on **Render**, set `CORS_ORIGINS = https://vertxing.vercel.app` (your exact
   Vercel URL, no trailing slash) and let it redeploy.
2. (If you later add a custom domain, append it comma-separated.)

## 8. Use it on your Android phone

1. Open `https://vertxing.vercel.app` in **Chrome** on Android.
2. **Register** — the first account becomes **Super-admin**.
3. Chrome menu → **Install app / Add to Home Screen**.
4. Open the installed app → **Settings → Background calls → Enable** → allow
   notifications.
5. Register a second account (another phone/browser), give it **call access** in
   **Settings → Administration → Users & Roles**, and call between them — including
   with the app **closed**.

## Optional: always-on & 100% free (Oracle Cloud)

To avoid the free-tier sleep, run the same Docker image on an **Oracle Cloud
"Always Free" ARM VM** (4 vCPU / 24 GB, free forever):

1. Create an **Always Free Ampere (ARM) VM** (Ubuntu), open ports 80/443.
2. Install Docker, point a free hostname at it (e.g. **DuckDNS**), and put
   **Caddy** in front for automatic HTTPS.
3. Build & run the API image:
   ```bash
   docker build -f apps/api/Dockerfile -t vertxing-api .
   docker run -d --restart unless-stopped -p 4000:4000 --env-file apps/api/.env.production vertxing-api
   ```
4. Keep **Neon + Upstash + LiveKit Cloud** as above (or self-host Postgres/Redis on
   the same VM). Point Vercel's `NEXT_PUBLIC_API_URL` and the API's `CORS_ORIGINS`
   at your domain.

This is more setup but never sleeps and costs nothing.

## Notes
- **Background push needs HTTPS** — that's why phone testing requires the deployed
  URL (not `http://…:3000`). Localhost works only on desktop.
- The **LiveKit media URL is provided by the API** in the join/accept response, so
  the web doesn't need a `NEXT_PUBLIC_LIVEKIT_URL`.
- Reliability on some Android OEMs (Xiaomi/Oppo/Vivo) depends on the user allowing
  notifications and disabling battery optimization for the installed app.
